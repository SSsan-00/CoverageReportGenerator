# ユーザー向けガイド

CoverageReportGenerator は、dotCover が出力した DetailedXML と対象プロジェクトの `.csproj` を読み込み、HTML / Excel カバレッジレポートを作成する Windows アプリです。

## できること

- プロジェクト全体のカバレッジ確認
- Namespace / Type / メンバー / ファイル単位のカバレッジ確認
- カバレッジが低い箇所のランキング表示
- メンバー名クリックによる対象ソース行へのジャンプ
- 選択したファイルの Excel レポート出力
- Excel で未カバー行を `※` 付きで確認
- Shift_JIS / CP932 ソースの日本語表示

## 入力に必要なもの

### `.csproj`

解析対象の C# プロジェクトファイルです。Razor Pages プロジェクトを主対象にしています。

アプリは `.csproj` を起点にソースファイル一覧を取得します。フォルダだけを直接読み込むのではなく、必ず先に `.csproj` を選択してください。

### dotCover DetailedXML

dotCover で生成した詳細 XML レポートです。HTML レポートではなく `DetailedXML` 形式を指定してください。

## HTML レポートの作り方

1. アプリを起動します。
2. `プロジェクト` の `選択` から `.csproj` を選択します。
3. `DotCover XML` の `選択` から DetailedXML を選択します。
4. `対象範囲` を選択します。
5. `出力先` と `レポート名` を確認します。
6. `HTML生成` を押します。

生成後に開く設定が有効な場合、作成された HTML が自動で開きます。

## 対象範囲

### プロジェクト

`.csproj` 配下の対象ソース全体をレポートします。

`対象範囲` を `プロジェクト` にすると、`対象パス` は空欄になり、入力不可になります。この状態では検出された対象ソース全体が出力対象です。

### フォルダ

指定したフォルダ配下を再帰的に対象にします。

`.csproj` 読み込み後、対象プレビュー左側にフォルダツリーが表示されます。ツリーのフォルダをクリックすると、対象範囲が `フォルダ` に切り替わり、`対象パス` にフォルダのフルパスが入ります。

ツリーのルートをクリックすると、対象範囲は `プロジェクト` に戻ります。

### ファイル

指定した単一ファイルだけを対象にします。

対象プレビュー右側のファイル一覧で行をクリックすると、対象範囲が `ファイル` に切り替わり、`対象パス` にファイルのフルパスが入ります。

## 対象プレビュー

`.csproj` 読み込み後、対象プレビューには次の情報が表示されます。

- 左側: フォルダツリー
- 右側: ファイル一覧

ファイル一覧の上部には絞り込み入力があります。ファイル名、フォルダ名、拡張子、フルパスの一部を入力すると、表示されるファイル一覧だけが絞り込まれます。スペース区切りで複数語を入力した場合は、すべての語に一致するファイルを表示します。

絞り込みはプレビュー表示だけに作用します。実際の出力対象は `対象範囲` と `対象パス` で決まります。

## 暗黙の対象フィルタ

画面上に Include / Exclude 入力欄はありません。

対象ソースは内部で次の条件を使って選択します。

```text
Include: *.cs;*.cshtml
Exclude: *.g.cs;*.generated.cs;*.Designer.cs;bin;obj
```

生成コードや `bin` / `obj` 配下は暗黙に対象外です。

## HTML レポートの見方

### 概要

レポート名、プロジェクト、対象範囲、全体カバレッジを表示します。カバレッジ率はバーでも表示されるため、数値だけでなく視覚的に状態を把握できます。

### ランキング

低カバレッジの Namespace / Type / メンバーと、未カバーが多いファイルを表示します。

### メンバー

メソッド、コンストラクタ、プロパティ、アクセサ、ローカル関数、ラムダなどを一覧表示します。メンバー名をクリックすると、`ソース` タブの該当行へジャンプします。

### ファイル

ファイル単位のカバレッジを一覧表示します。

### ソース

ソース行をカバレッジ状態付きで表示します。

- `C`: Covered
- `U`: Uncovered
- 空欄: No Data

dotCover 公式 HTML と行ハイライトが一致するよう、複数行 Statement や同一行の混在ケースを考慮しています。

## Excel レポートの作り方

1. `.csproj` と DotCover XML を選択します。
2. プロジェクトが読み込まれていることを確認します。
3. `Excel出力` を押します。
4. ファイル選択ダイアログで対象ファイルを追加します。
5. 保存先を指定します。

ファイル選択ダイアログでは、ファイル名、フォルダ名、拡張子で絞り込みできます。左側の一覧からファイルを選んで `追加` すると、右側の選択済みリストへ入ります。不要なファイルは選択済みリストから削除できます。

## Excel レポートの見方

Excel レポートは選択したファイルごとにシートを作成します。1ファイルだけ選択した場合は従来どおり1シートのブックになります。

各シートの上部には対象ファイル、カバレッジ率、Statement 数を表示します。メンバー一覧では、各メンバーのカバレッジ率を色とバーで確認できます。

メンバー名をクリックすると、同じシート内の該当ソース定義行へジャンプします。文字列検索ではなく Roslyn 解析で得た定義開始行を使うため、同じ名前の呼び出し行ではなく定義行へ移動します。

ソース一覧は次の列構成です。

```text
A列: 行番号
B列: 未カバー印
C列: 本文
```

未カバー行の B 列には `※` を表示します。未カバーでない行の B 列には値を書き込まないため、Excel の `Ctrl + ↑` / `Ctrl + ↓` で `※` のあるセルへ移動しやすくなっています。

Excel シートのヘッダー固定とオートフィルターは設定していません。メンバー数が多いファイルでも、ソース本文を見やすくするためです。

## 保存される設定

前回入力した内容は次回起動時に復元されます。

保存対象:

- プロジェクトパス
- DotCover XML パス
- 対象範囲
- 対象パス
- 出力先
- レポート名
- 生成後に開く
- 既存ファイル上書き

初期状態に戻したい場合は `初期化` ボタンを押します。

## キャッシュ

`.csproj` のソース検出結果と Roslyn のメンバー解析結果は、次の場所にキャッシュされます。

```text
%LOCALAPPDATA%\CoverageReportGenerator\cache
```

ソースが変更された場合は必要な分だけ更新します。表示がおかしい場合は、アプリの初期化またはキャッシュ削除後に再読み込みしてください。

## Bootstrap で導入する

リポジトリを clone できない場合は、単一 PowerShell ファイルで導入できます。

```powershell
.\bootstrap\CoverageReportGenerator.Bootstrap.ps1 -Output .\dist
```

出力先には `CoverageReportGenerator.exe` と `source/` が作成されます。`source/` にはアプリ本体、テストコード、ドキュメントを含みます。

bootstrap の成果物には `bootstrap/` は含まれません。`source/` にコピーされる `README.md` と `docs/*.md` からは、Bootstrap に関する章と説明行を省きます。

## よくあるトラブル

### Target Preview の件数が少ない

次を確認してください。

- `.csproj` が正しいか
- 対象範囲がフォルダや単一ファイルに絞られていないか
- ファイル一覧プレビューの絞り込み入力に文字が残っていないか
- Razor Pages の `.cshtml.cs` がプロジェクトに含まれているか

### 文字化けする

HTML は UTF-8 BOM 付きで出力します。ソースファイルは UTF-8、UTF-16、Shift_JIS / CP932 を読み取ります。

それでも文字化けする場合は、対象 XML またはソースファイルの実際のエンコーディングを確認してください。

### HTML のカバレッジ行が dotCover と違う

このツールは dotCover DetailedXML の Statement 情報を元に行表示します。dotCover のバージョンや XML 形式が変わった場合は差が出る可能性があります。

差分がある場合は、dotCover の HTML レポート、DetailedXML、対象ソースをセットで確認してください。

### Excel のリンクが呼び出し行に飛ばないか心配

Excel のメンバーリンクは、メンバー名の文字列検索ではなく Roslyn 解析で得た定義開始行を使います。同じ名前の呼び出しがソース内にあっても、定義行へジャンプします。

### レポート生成に失敗する

次を確認してください。

- DotCover XML が DetailedXML 形式か
- XML 内に `FileIndices/File` と `Statement` が含まれているか
- XML のファイルパスと `.csproj` 配下のソースパスが対応しているか
- 出力先フォルダに書き込み権限があるか
